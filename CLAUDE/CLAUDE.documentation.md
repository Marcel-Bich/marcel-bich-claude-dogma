# Documentation Rules

<rules>
1. **README required** - Every project needs a README with setup instructions.
2. **Comments explain why** - Code says what, comments say why (if not obvious).
3. **No obvious comments** - Don't comment self-explanatory code.
4. **Update with code** - Change comments when changing code.
5. **JSDoc for public APIs** - Document exported functions/classes.
</rules>

For details see `GUIDES/documentation.md`
